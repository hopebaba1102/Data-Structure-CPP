Chapter 0: Release Notes

The files compid.h and dslib.h are designed to make all the software in the portable by minimizing dependencies on particular compilers. All example code in the book includes dslib.h. dslib.h in turn includes compid.h. The idea is that in order to add support for a new compiler, you should edit the compid.h and leave dslib.h alone. Typically, your compiler will have one or more predefined variable that will uniquely identify the compiler, and often the version number. Take a look at compid.h and you can see how it works.

Right now, the following preprocessor variables are used by dslib.h:

  • __MB_COMPILER_DEFINES_BOOL__: this should be set if the compiler you are using implements bool, true, and false; otherwise, dslib.h will use an enum to fake it.
  • __MB_SIMULATE_BORLAND_RANDOM__: set this for any compiler that does not implement random numbers in the style used by the Borland compilers.

Originally, I was going to support some of the older compilers, such as Turbo C++ 3.0, using this mechanism, but it made the header files very messy. Instead, take a look at Notes on Specific Compilers for information on Turbo C++ 3.0 and other compilers that have particular issues. In some cases, you will need to modify the code by hand.
